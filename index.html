What is a Switch Statement?

A switch statement is a control flow statement that evaluates an expression, matching the expression's value against a series of case clauses, and executes statements after the first case clause with a matching value.

It provides an efficient way to select one of many code blocks to be executed.

When to Use Switch Statements

Use switch statements when you have:

  • A single expression to evaluate against multiple possible values
  • Many potential conditions that would otherwise require multiple if-else statements
  • A need for cleaner, more readable code compared to many if-else statements
  • Values to compare are of the same type and have defined, discrete values

Switch vs. If-Else

If-Else Structure


if (fruit === 'apple') {
    console.log('Apples are $0.99 per pound');
} else if (fruit === 'banana') {
    console.log('Bananas are $0.59 per pound');
} else if (fruit === 'orange') {
    console.log('Oranges are $0.79 per pound');
} else {
    console.log('Sorry, we are out of ' + fruit);
}

Switch Structure


switch (fruit) {
    case 'apple':
        console.log('Apples are $0.99 per pound');
        break;
    case 'banana':
        console.log('Bananas are $0.59 per pound');
        break;
    case 'orange':
        console.log('Oranges are $0.79 per pound');
        break;
    default:
        console.log('Sorry, we are out of ' + fruit);
}

Key Differences

  • Readability: Switch statements can be more readable when comparing a single variable against multiple values
  • Evaluation: If-else can evaluate any boolean expression; switch compares a single expression against case values
  • Fallthrough: Switch cases fall through (continue to the next case) if there's no break statement
  • Default: Switch has a built-in default case for handling when no matches occur

Syntax and Structure


switch (expression) {
    case value1:
        // Code to execute when expression === value1
        break;
    case value2:
        // Code to execute when expression === value2
        break;
    case value3:
    case value4: // Multiple cases can share the same code block
        // Code to execute when expression === value3 OR value4
        break;
    default:
        // Code to execute if none of the cases match
}

Key Components

1. switch (expression)

The expression to evaluate. Can be a variable, function call, or any expression that returns a value.

2. case value:

Each case label specifies a value to compare with the expression. If they match, the code after the case label executes.

3. break;

Exits the switch statement. Without break, execution "falls through" to the next case.

4. default:

Optional block that executes if none of the case values match the expression.

Example 1: Basic Switch Statement

Day of the Week


const day = 3; // Let's assume 1 = Monday, 2 = Tuesday, etc.
let dayName;

switch (day) {
    case 1:
        dayName = 'Monday';
        break;
    case 2:
        dayName = 'Tuesday';
        break;
    case 3:
        dayName = 'Wednesday';
        break;
    case 4:
        dayName = 'Thursday';
        break;
    case 5:
        dayName = 'Friday';
        break;
    case 6:
        dayName = 'Saturday';
        break;
    case 7:
        dayName = 'Sunday';
        break;
    default:
        dayName = 'Invalid day';
}

console.log(dayName); // Outputs: "Wednesday"

Key Points

  • The expression day is compared against each case value
  • When a match is found (case 3), the code after that case executes
  • The break statement prevents fall-through to other cases
  • If day didn't match any case, the default case would execute

Example 2: Switch with Strings

Fruit Prices


const fruit = 'banana';
let price;

switch (fruit.toLowerCase()) { // Convert to lowercase for case-insensitive comparison
    case 'apple':
        price = 0.99;
        break;
    case 'banana':
        price = 0.59;
        break;
    case 'orange':
        price = 0.79;
        break;
    case 'mango':
    case 'papaya': // Multiple cases with the same action
        price = 1.29;
        break;
    default:
        price = 'Price not available';
}

console.log(`${fruit} costs $${price} per pound`); // Outputs: "banana costs $0.59 per pound"

Key Points

  • Switch statements work with string values
  • The expression can include method calls (fruit.toLowerCase())
  • Multiple case values (mango, papaya) can share the same code block
  • Case comparisons are strict (===), so type matters

Example 3: Fallthrough Behavior

Days in Month


function getDaysInMonth(month, year) {
    let days;
    
    switch (month) {
        case 1: // January
        case 3: // March
        case 5: // May
        case 7: // July
        case 8: // August
        case 10: // October
        case 12: // December
            days = 31;
            break;
            
        case 4: // April
        case 6: // June
        case 9: // September
        case 11: // November
            days = 30;
            break;
            
        case 2: // February
            // Check for leap year
            if ((year % 4 === 0 && year % 100 !== 0) || year % 400 === 0) {
                days = 29;
            } else {
                days = 28;
            }
            break;
            
        default:
            days = 'Invalid month';
    }
    
    return days;
}

console.log(getDaysInMonth(2, 2024)); // Outputs: 29 (2024 is a leap year)

Key Points

  • Intentional fallthrough is useful for case groups that share the same output
  • Each case group ends with a break to prevent unintended fallthrough
  • Complex logic (leap year calculation) can be included within case blocks
  • Return statements also exit the switch, even without break

Example 4: Default Case & Common Patterns

Grade Calculator


function getGradeFeedback(grade) {
    let feedback;
    
    switch (true) { // Evaluating expressions with true allows for range checking
        case grade >= 90:
            feedback = 'Excellent! You got an A.';
            break;
        case grade >= 80:
            feedback = 'Good job! You got a B.';
            break;
        case grade >= 70:
            feedback = 'You got a C. Keep working!';
            break;
        case grade >= 60:
            feedback = 'You got a D. You need to study more.';
            break;
        case grade >= 0:
            feedback = 'You failed with an F. Please seek help.';
            break;
        default:
            feedback = 'Invalid grade entered.';
    }
    
    return feedback;
}

console.log(getGradeFeedback(85)); // Outputs: "Good job! You got a B."

Advanced Switch Patterns

  • Using switch(true): Allows evaluating ranges by checking which case condition returns true first
  • Default placement: Can be placed anywhere in the switch, but conventionally at the end
  • Early returns: Using return statements inside cases eliminates the need for break
  • Validation: Default case is useful for input validation and error handling

Common Pitfalls & Best Practices

Forgetting Break Statements


// Problem: Missing break statements
switch (color) {
    case 'red':
        console.log('Stop!');
        // Missing break - will fall through to the next case
    case 'yellow':
        console.log('Caution!');
        // Missing break - will fall through to the next case
    case 'green':
        console.log('Go!');
        break;
}

// If color is 'red', this will output:
// "Stop!"
// "Caution!"
// "Go!"

Best Practices

  • Always include break statements unless fallthrough is intentional
  • Comment intentional fallthrough cases to make your intention clear
  • Consider using object literals or maps for simple value mappings instead of switch
  • Always include a default case to handle unexpected inputs
  • Keep case statements simple; move complex logic to separate functions

Alternative Pattern: Object Lookup


// Instead of switch, you can use object lookup for simple mappings
const fruitPrices = {
    apple: 0.99,
    banana: 0.59,
    orange: 0.79,
    mango: 1.29,
    papaya: 1.29
};

const price = fruitPrices[fruit.toLowerCase()] || 'Price not available';
console.log(`${fruit} costs $${price} per pound`);

Question 1

What happens if you omit the break statement in a switch case?

A. The switch statement will throw an error

B. The switch statement will automatically exit

C. The code will continue to execute the next case

D. The default case will always execute

Correct! Without a break statement, execution will "fall through" to the next case, regardless of whether that case matches the expression.

Question 2

Which of the following can be used as the expression in a switch statement?

A. Strings and numbers only

B. Any expression that returns a value

C. Only boolean expressions

D. Only variables, not function calls

Correct! A switch statement can evaluate any expression that returns a value, including variables, function calls, or arithmetic operations.

Question 3

What is the purpose of the default case in a switch statement?

A. It's required syntax and must be included in every switch statement

B. It executes only if all other cases have been executed

C. It executes when none of the case values match the switch expression

D. It specifies the default value of the switch expression

Correct! The default case executes when none of the case values match the expression being evaluated. It's optional but recommended for handling unexpected inputs.

Question 4

What type of comparison does switch use when matching cases?

A. Loose equality (==)

B. Strict equality (===)

C. Greater than or equal to (>=)

D. It doesn't use comparison operators

Correct! Switch statements use strict equality (===) when comparing the expression with case values, meaning both value and type must match.

Question 5

Which of the following accurately describes a potential disadvantage of switch statements?

A. They can only handle a maximum of 10 cases

B. They are always slower than if-else statements

C. They only work with primitive data types

D. Forgetting break statements can lead to unexpected behavior

Correct! Forgetting to include break statements can lead to unintentional fallthrough, causing multiple case blocks to execute when only one was intended.

Key Takeaways

  • Switch statements provide a clear way to handle multiple conditions for a single variable
  • Always include break statements unless fallthrough is intentional
  • The default case handles unexpected inputs and is a best practice
  • Switch uses strict equality (===) when comparing values
  • For simple value mappings, consider object literals as an alternative
